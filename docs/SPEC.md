# Action Biome 仕様書

## ゲーム概要

本作は Godot で制作する 2D サイドビューのアクションサンドボックスゲームである。

プレイヤーは墜落した宇宙船を起点に、上下左右へ広がる有限ワールドを探索する。ゲームの基本体験は、スーパーマリオのような横スクロールアクションと、テラリアのようなサイドビューサンドボックスを組み合わせたものとする。

ワールドはシード値によって生成され、複数のバイオーム、モンスター、部屋、クラフトによる成長要素を持つ。最初に入ったワールドをメインワールドとし、以後は複数のサブワールドをまたいで旅をする。

## コンセプト

- アクションゲームとして、ジャンプ、ダッシュ、攻撃、踏みつけによる撃破などの手触りを重視する。
- サンドボックスゲームとして、ワールド探索、部屋作成、調度品配置、クラフトによる宇宙服強化を重視する。
- SF要素として、墜落した宇宙船、宇宙服、ロボット型モンスター、エイリアン型モンスター、ワールド間移動を扱う。
- 明るいアクションゲームを参考にし、リアル寄りではなくデフォルメされた表現にする。
- バイオームの種類が多いことをゲームの特徴にする。

## ビジュアル方針

- ドット絵、ピクセルアートを基本とする。
- レトロな雰囲気を重視する。
- モンスターやオブジェクトは、写実的ではなくデフォルメされた見た目にする。

## プレイヤー開始地点と宇宙船

- ゲーム開始地点は、ワールド中心に墜落した宇宙船とする。
- 宇宙船から上下左右にワールドが広がる。
- 宇宙船は破壊されない特別な建物として扱う。
- 宇宙船は初期拠点として、寝室と作業台からなる基本的な部屋を持つ。
- 宇宙船には操縦室があり、他ワールドへのワープポイントになる。

## ワールド仕様

- ワールドはシード値によって生成される。
- ワールドサイズは有限とする。
- プレイヤーは単一ワールド内だけで完結せず、複数ワールドをまたいで旅をする。
- 最初に入ったワールドをメインワールドとする。
- 追加ワールドはサブワールドとして扱う。
- サブワールドは複数のシード値を保持し、宇宙船の操縦室から行き来できる。
- ワールド内の難易度は、中心の宇宙船からの距離に応じて上がる。

## バイオーム仕様

- ワールドは複数のバイオームで構成される。
- バイオーム同士の境界は厳密な線ではなく、緩やかに遷移する。
- 緩やかな遷移を前提とするため、隣接できないバイオームの組み合わせが存在する。
- 初期バイオームは平地とする。
- バイオームごとに出現するモンスターが決まる。
- バイオームに特徴的な部屋が、廃棄された壊れた状態で生成されることがある。

- 岩山

- 廃棄された都市
- 廃棄された工場地帯
- 古代遺跡
- 廃棄された実験室

## アクション仕様

- プレイヤーはジャンプ、ダッシュ、攻撃を行える。
- 敵は踏むことで倒せる場合が多い。
- 宇宙服にクラフトでオプションを追加し、プレイヤーを強化していく。
- 強化内容は宇宙服の機能追加として表現する。

## モンスター仕様

- モンスターはバイオームごとに決まる。
- 中心の宇宙船から遠いほど、モンスターは強くなる。
- モンスターの見た目は、明るいアクションゲームを参考にしたデフォルメ表現とする。
- SF要素として、ロボット型やエイリアン型のモンスターを含めてよい。
- 中ボスは将来的な実装候補とするが、初期リリースでの扱いは未定とする。

## 部屋仕様

- プレイヤーが閉空間を作成すると、そこを部屋として扱う。
- 部屋に配置された調度品によって、部屋の種類とレベルが決まる。
- 体力は寝室にいると自然回復する。
- 食料は食堂で調理して食べられる。
- 扉を使うと、閉空間の中と外を出入りできる。
- モンスターは基本的には部屋に入ってこない。
- 部屋に入ってくるモンスターの例外は存在し得る。
- バイオームに特徴的な部屋が、廃棄された壊れた状態で生成されることがある。

### 部屋案

- 寝室: ベッドとチェスト
- 作業場: 作業台とチェスト
- 食堂: キッチン、テーブル、椅子
- 教会: 祭壇とベンチ
- 図書館: 本棚、机、椅子
- 武器庫: 武器ラックとチェスト

## 宇宙船仕様

- 宇宙船はワールド中心に墜落している。
- 宇宙船は破壊されない。
- 宇宙船は基本拠点として、寝室と作業台を含む。
- 宇宙船には操縦室という特別な部屋がある。
- 操縦室は他ワールドへのワープポイントとして機能する。
- 最初に入ったワールドをメインワールドとして扱う。
- サブワールドのシード値を複数保持し、ワールド間を行き来できる。

## スポーン・停止・デスポーン方針

- 動くものは、主人公から一定以上離れると停止する。
- 停止対象は、画面外であっても主人公からの距離を基準にする。
- オブジェクトによってはデスポーンする。
- スポーン数は、実装後の動作に支障が出ない現実的な範囲で決める。
- 可能な限り制約を強くしすぎず、多様な状況を許容する。

## AI実装向け仕様

このセクションは、AIエージェントに初期実装を任せるための基準とする。ゲームデザイン上の理想よりも、最初に遊べる縦スライスを完成させることを優先する。

### 実装前提

- Godot 4系を使用する。
- スクリプトは GDScript を使用する。
- ゲームは 2D サイドビューとする。
- ワールドはタイル地形ベースで実装する。
- 1タイルは 32px とする。
- 内部解像度は 640x360 とする。
- 初期入力はキーボード+マウスを対象にする。
- 外部素材はデフォルトでは使用しない。使用する場合は、ライセンス、入手元、使用範囲を明記する。

### 初期実装の到達点

初期実装は縦スライスとする。完成判定は、短時間でもゲーム開始から探索、採掘、配置、クラフト、戦闘、部屋作成、死亡復帰、セーブ/ロードまで一通り確認できる状態とする。

- 勝利条件は置かず、自由探索を目的にする。
- ワールドサイズは 512x256 タイル程度を初期値とする。
- 初期バイオームは平地、森林、洞窟の3種類とする。
- ワールド間移動は、初期では宇宙船の操縦室UIと入口まで実装する。
- 実際に別シードのサブワールドへ移動する機能は後続対象とする。
- 中ボス、本格マップ、図鑑、大規模インベントリ管理は後続対象とする。

### 初期操作

- A/D: 左右移動。
- Space: ジャンプ。
- Space長押し: 可変ジャンプ。
- Shift: 短距離ダッシュ。
- 左クリック: 攻撃または採掘。
- 右クリック: ブロック配置またはオブジェクト使用。
- E: インタラクト。
- 数字キー: 選択スロット切り替え。
- F3: デバッグモード切り替え。

### プレイヤー

- 操作感は軽快なアクション寄りにする。
- 移動、ジャンプ、可変ジャンプ、コヨーテ時間、ダッシュ、攻撃を実装する。
- 敵を踏みつけた場合、倒せる敵は撃破される。
- 攻撃、採掘、配置はマウス位置または選択中タイルを対象にする。
- 死亡時は所持品を維持し、宇宙船に復帰する。
- 死亡復帰時、HPは初期値まで回復する。
- デバッグモード中は、プレイヤーのHPが減らず死亡しない。現時点のデバッグモード効果はHP減少無効のみとし、将来的に他の検証支援機能を追加できるものとする。

### 初期調整値

以下は初期実装用の仮調整値であり、実装後のプレイ感に応じて変更してよい。

- プレイヤー最大HP: 5。
- 移動速度: 220px/s。
- ジャンプ初速: 420px/s。
- 重力: 1200px/s^2。
- コヨーテ時間: 0.10秒。
- ダッシュ速度: 420px/s。
- ダッシュ時間: 0.16秒。
- 攻撃/採掘射程: 4タイル。
- 自動保存間隔: 60秒。
- 主人公から 40タイル以上離れた動的オブジェクトは停止候補とする。
- 同時アクティブ敵数は 30体程度を初期上限とする。

### 採掘・配置・クラフト

- プレイヤーはタイル地形を採掘し、素材を入手できる。
- プレイヤーは入手した素材を使ってブロックや家具を配置できる。
- クラフトは作業台から行う。
- 初期クラフトは簡易素材循環に留め、多段階の複雑なレシピ網は作らない。

初期素材:

- 土
- 石
- 木材
- 鉱石
- 植物素材

初期家具:

- 作業台
- 扉
- チェスト
- ベッド
- テーブル
- 椅子
- キッチン

初期宇宙服強化:

- ダッシュ強化
- ジャンプ強化
- 採掘強化

### 部屋判定

- 初期の部屋判定は単純閉空間とする。
- 壁、床、天井、扉で囲まれた空間を部屋候補とする。
- 部屋種別は配置された家具で決定する。
- 寝室はベッドとチェストを必要家具とする。
- 作業場は作業台とチェストを必要家具とする。
- 食堂はキッチン、テーブル、椅子を必要家具とする。
- 部屋レベル計算は初期では簡易にし、必要家具が揃っていればレベル1とする。

### モンスター最小構成

- 平地には、踏みつけ可能な小型敵を配置する。
- 森林には、ジャンプまたは浮遊で動くデフォルメ敵を配置する。
- 洞窟には、ロボット型またはエイリアン型の強めの敵を配置する。
- 各敵は待機、移動、被弾、消滅の最低限の状態を持つ。
- 各敵は最低限のスプライトアニメーションを持つ。
- 中心の宇宙船から遠いほど、敵のHP、攻撃力、出現頻度のいずれかが上がる。

### アセット方針

- 初期実装から、AI生成ピクセルアートをリリースレベル品質寄りで作成する。
- 仮矩形や単色記号のみの見た目は、初期完成状態として扱わない。
- 量よりも、後続追加時の指針になる品質と統一感を優先する。
- 色数は抑えめにし、各キャラクターとオブジェクトに読みやすいシルエットを持たせる。
- 主要スプライトにはアウトラインと簡単な影を入れる。
- アニメーションは少ないフレーム数でもよいが、待機、移動、使用、被弾などの状態差を表現する。
- ファイル名は対象、状態、連番が分かる命名にする。

初期アセットの必要最小セット:

- プレイヤー
- 平地、森林、洞窟の3敵
- 平地、森林、洞窟のタイル
- 採掘/配置可能なブロック
- 初期家具
- 墜落した宇宙船
- 操縦室
- HUDと簡易メニュー用UI

### セーブ仕様

- セーブは自動保存と手動保存の両方を実装する。
- 手動保存は宇宙船またはメニューから行える。
- 自動保存は一定間隔で行う。
- 保存対象は、プレイヤー位置、HP、所持品、宇宙服強化、地形変更、部屋情報、ワールドseedとする。
- 初期実装では、別シードのサブワールド状態保存は後続対象とする。

### UI範囲

- HUDにはHP、素材数、選択スロットを表示する。
- 作業台では簡易クラフトメニューを開ける。
- 操縦室ではワールド間移動入口のUIを開ける。
- 操縦室UIには、初期実装ではサブワールド移動が後続対象であることが分かる状態を表示する。
- セーブ時は保存完了が分かる表示を出す。
- 本格マップ、図鑑、大規模インベントリ管理は後続対象とする。

### 実装チェックリスト

- 起動直後に宇宙船からプレイ開始できる。
- 平地、森林、洞窟を探索できる。
- タイル地形を採掘できる。
- ブロックまたは家具を配置できる。
- 作業台で初期クラフトができる。
- 寝室、作業場、食堂を作成し、部屋として判定できる。
- 敵を攻撃または踏みつけで倒せる。
- 死亡時に所持品を維持して宇宙船へ復帰できる。
- 自動保存と手動保存が機能する。
- セーブデータからロードできる。
- 操縦室UIを開ける。
- 初期アセットが仮矩形ではなく、方向性を示すピクセルアートになっている。

## 初期リリース対象

初期リリースでは、以下を主要対象とする。

- Godot 製の 2D サイドビューアクションサンドボックスとして遊べること。
- 墜落した宇宙船から探索を開始できること。
- 512x256タイル程度の有限サイズのシード生成ワールドを探索できること。
- 平地、森林、洞窟を持つワールドを生成できること。
- ジャンプ、可変ジャンプ、コヨーテ時間、ダッシュ、攻撃などの基本アクションがあること。
- 踏みつけで倒せる敵が存在すること。
- バイオームごとにモンスターが分かれること。
- 宇宙船からの距離に応じてモンスターが強くなること。
- プレイヤーが閉空間を作り、部屋として扱えること。
- 寝室、作業場、食堂など、基本的な部屋の役割があること。
- 宇宙船を破壊されない拠点として扱えること。
- 操縦室をワールド間移動の入口として扱えること。
- 自動保存と手動保存ができること。
- AI生成のピクセルアートアセットで、初期の見た目の方向性が確認できること。

- バイオーム遷移の具体的な生成方式。
- モンスター強化に使う宇宙船からの距離計算方式。
- 中ボスを初期リリースに含めるかどうか。
- 部屋に入れるモンスター例外の条件。
- デスポーン条件。
- 複数サブワールドの本実装。
- 本格マップ、図鑑、大規模インベントリ管理。
- 詳細な敵名、素材名、レシピ名、家具名。
- 詳細なゲームバランス値。
